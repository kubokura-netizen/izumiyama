# 🛠 ツール改変 依頼テンプレート（Claude Code へ貼り付けて使う）

このツール（PRP申請書類 自動転記ツール）を**壊さずに安全に改変**するための指示書です。
プログラミングの知識がなくても、下の「**■ Claudeへの指示**」を丸ごとコピーして、
**【変更内容】** に自分のやりたいことを書き、**Claude Code に貼り付ける**だけで改変できます。

---

## 📌 使い方（3ステップ）

1. 下の「**■ Claudeへの指示（ここからコピー）**」〜「**（ここまでコピー）**」を**丸ごとコピー**。
2. `【変更内容】` の下に、**やりたい変更を日本語で具体的に**書く（例文は後述）。
3. このツールのフォルダを開いた **Claude Code** に貼り付けて送信。

> 💡 Claudeが自動で「今の作りを調べる → バックアップ → 変更 → テスト実行して確認」まで行います。
> 変更後は必ず `転記実行.bat` を一度実行し、`02_output` の中身を目視で確認してください。

# 付録A：ツールの全体像（Claudeが理解するための地図）

## フォルダ構成
```
PRP申請書類 自動転記ツール/
├─ 転記実行.bat                         … 実行入口（py transcribe.py を呼ぶ）
├─ 00_マニュアル/                        … 本書・操作マニュアル
├─ 01_input【…】/                        … ヒアリングシート(.xlsx)を置く
├─ 02_output【…】/                       … 出力（フォルダ単位で生成）
├─ 03_logs/                             … 転記ログ
└─ 99_data/
   ├─ テンプレート/
   │   ├─ 2種関節系PRP/                 … 様式xlsx＋各docx＋PDF一式
   │   ├─ 3種筋腱靭帯系PRP/
   │   ├─ SOP/                          … Word手順書16本＋教育研修記録xlsx
   │   └─ _backup_v3.0/                 … 改変前バックアップ（増やしてよい）
   ├─ マッピング/
   │   ├─ docs_config.json              … ★転記設定の本体（最重要）
   │   ├─ mapping.json                  … 様式セル転記の元定義（docs_config生成の参照元）
   │   ├─ format_changes.json           … 様式変更の動的上書き（任意）
   │   └─ sop_replacements.json         … 旧SOP用（現在は未使用）
   ├─ src/
   │   ├─ transcribe.py                 … main。ヒアリング読込→run_docs呼出→ログ
   │   ├─ transcribe_docs.py            … ★フォルダ型エンジン（様式Excel＋docx置換＋医師/略歴書/キット）
   │   ├─ transcribe_sop.py             … Word置換の部品（段落/表/テキストボックス/複数行差込）
   │   └─ README_メンテナンス.md
   ├─ パッチノート/                       … 変更履歴（改変したら追記）
   └─ 参考元【原本・設計書】/
```

## 処理の流れ
`転記実行.bat` → `transcribe.py main()` → ヒアリング読込 → `transcribe_docs.run_docs()`：
各テンプレフォルダ（2種/3種/SOP）を `02_output/(フォルダ名)_(ヒアリング名)/` へコピーし、
1. **様式第一の二(.xlsx)** … `docs_config` の `excel.entries`（セル→ヒアリング項目）でセル転記＋
   `doctor_fill`（医師を最大15名、ブロックを動的検出して転記）。
2. **各docx** … `docs_config` の `docx`（サンプル値→ヒアリング値の一括置換）＋
   `docx_kit`（`{{採取方法}}`/`{{加工方法}}` を①②③連番展開）＋
   `docx_table`（略歴書を表の行ごとに転記）。
3. **PDF・論文・追加xlsx** … 無変更で同梱。

## docs_config.json の構造（抜粋）
```jsonc
{
  "hearing_sheet": "ヒアリングシート（PRP）",
  "documents": {
    "2種関節系PRP": {
      "folder": "2種関節系PRP",
      "excel": {
        "file": "01.【様式第一の二】…(PRP二種）.xlsx",
        "sheet": "1 の２提供計画（治療）",
        "entries": [ { "cell":"N11","var":"医療機関_名称","source":{…},"tf":"text" }, … ],
        "doctor_fill": { "block_label":"再生医療等を行う医師又は歯科医師",
                         "col":12,"name_offset":1,"affil_offset":2,"max":15 }
      },
      "docx_kit":   [ { "token":"{{採取方法}}","source":{"t":"kit","part":"採取","mode":"block","numbered":true} }, … ],
      "docx_table": [ { "file_contains":"略歴書","table":0,"value_col":1,
                        "rows":[ {"row":0,"source":{"t":"sheet","sheet":"略歴書（PRP）","cell":"B2"}}, … ] } ],
      "docx":       [ { "desc":"クリニック名","find":["やまだ整形外科クリニック", …],"source":{…} }, … ]
    },
    "3種筋腱靭帯系PRP": { … },
    "SOP": { "folder":"SOP","docx":[ … ] }
  }
}
```

## source（転記元）の種類
| t | 意味 | 例 |
|---|---|---|
| `hearing` | ヒアリングのラベル＋セクションで検索 | `{"t":"hearing","label":"…","section":"…"}` |
| `sheet` | ヒアリング内の別シートのセルを直接参照 | `{"t":"sheet","sheet":"略歴書（PRP）","cell":"B2"}` |
| `committee` | 委員会名→審査委員会シートで認定番号 | `{"t":"committee"}` |
| `kit` | PRPメーカーシートから採取/加工/投与。`mode:"block"`で①②③連番 | `{"t":"kit","part":"加工","mode":"block","numbered":true}` |
| `compose` | 複数項目を改行連結 | `{"t":"compose","lines":[…]}` |
| `doctor` | 医師（複数）※通常は doctor_fill を使う | `{"t":"doctor","idx":2,"field":"name"}` |
| `date` / `fixed` / `unknown` | 実行日 / 固定値 / 未定義 | |

# 付録B：変更タイプ別 対応早見表（Claudeが選ぶ指針）

> 補足：規制上の手続き（重要変更＝様式第2で委員会審査、軽微変更＝様式第3 等）は**使用者の申請業務**です。
> ここでは「**このツールの出力を変える**」ための改変方法を示します。

| 変更したいこと | 主な対応（設定優先） |
|---|---|
| **① 様式改訂で項目が追加**（例：科学的妥当性の評価方法の記載義務化） | テンプレ(.xlsx/.docx)に新項目欄を用意 → ヒアリングから転記が要るなら `docs_config` に転記設定を追加（excel.entries / docx / docx_table 等） |
| **② 様式改訂で項目が削除**（例：役職欄・審査受付番号欄の削除） | 該当の転記設定を削除/無効化（entryを消す・findを消す）＋テンプレから欄を削除 |
| **③ 医師の追加・変更** | ヒアリングの医師欄（C42以降）に追加すれば `doctor_fill` が最大15名自動転記。**テンプレの医師ブロックが足りなければ、様式Excelで医師ブロック（氏名・所属・役職の4行）をコピーして増やす** |
| **④ PRPキットの変更・追加** | キットはヒアリングの「PRPメーカー」シート由来。使うキットの選択はヒアリング側。**新種キットを選べるようにする**なら PRPメーカーシートに行を追加（A:名称/B:採血量/C:採取/D:加工/E:投与）。書類2のキット本文は `{{採取方法}}{{加工方法}}` トークンで自動連番展開される |
| **⑤ 提供価格の変更** | 価格はヒアリング/テンプレのテキスト。ヒアリング項目化して転記 or `docx` の find-replace or テンプレ直接編集 |
| **⑥ 軽微な文言変更**（電話番号・部署名・誤記） | `docs_config` の `docx`（find→source）に置換を追加、またはテンプレを直接編集 |
| **⑦ クリニック情報の初期値差し替え**（サンプル値を別クリニックに） | 各docxのサンプル値と `docx.find` を合わせる（例：やまだ整形外科クリニック→新サンプル） |

## 【変更内容】の書き方 例文
- 「様式第一の二に『科学的妥当性の評価方法』の欄が追加されたので、ヒアリングの『○○』項目から転記されるようにして。転記先セルは△△。」
- 「実施責任者と医師の『役職』欄が新様式で削除されたので、役職の転記をやめて。」
- 「PRPメーカーに『□□キット（△△社）』を追加したい。採血量○mL、採取・加工・投与の手順は以下（本文）。」
- 「説明書の電話番号 0463-59-9901 を、ヒアリングの実施責任者電話から入るようにして。」
- 「医師を最大15名まで実際に入れられるよう、2種・3種の様式に医師ブロックを15個に増やして。」

# 付録C：安全ルールと検証手順（Claude向けの詳細）

## バックアップ
- テンプレ改変前：`99_data/テンプレート/_backup_(YYYYMMDD)/` に対象フォルダをコピー。
- 設定/コードは git 管理があればコミット、無ければ改変前を `*.bak` で残す。

## 検証（変更後は必ず）
```
py 99_data/src/transcribe.py            # 01_input の最新ヒアリングで実行
py 99_data/src/transcribe.py "パス.xlsx"  # ファイル指定で実行
```
- **出力フォルダ名**が `2種関節系PRP_(ヒアリング名)` 等になっているか（壊れた長い名前＝変数衝突を疑う）。
- 変更項目が反映され、旧サンプル値（例：やまだ整形外科クリニック／さこ整形外科／山田太郎／迫浩輔／JSCSF 等）が残っていないか。
- **文字化け対策**：Windowsコンソールは cp932 で日本語が化ける。検証は
  - 結果を UTF-8 のテキストファイルに書き出して読む、または
  - docx は `word/*.xml` の `<w:t>…</w:t>` を正規表現で抽出した「可視テキスト」で数える
  （※ `<w:t>` 以外のXML属性(paraId等)に 000000000 等が現れ過剰カウントするので、必ず可視テキストで判定）。

## よくある落とし穴
- **セル座標のズレ**：2種と3種は行構成が1行ずれる箇所がある（例：医師氏名は 2種=L60 / 3種=L58、委員会は 2種=L155 / 3種=L152）。種ごとに確認。
- **置換対象を割らない**：テンプレの「置換される値」や `{{トークン}}` の**途中に改行・スペースを入れない**（完全一致でマッチするため）。周囲の体裁調整は自由。
- **テキストボックス内の文字**：Wordの図形内テキストは通常の段落APIで届かない（低レベル `<w:t>` 走査で対応済み。新規で触るときは注意）。
- **変数名の衝突**：`transcribe_docs.py` でファイル名は `fbase`、ヒアリング名は `base`。混同すると出力フォルダ名が壊れる（過去の不具合）。
- **医師ブロック不足**：医師数 > テンプレのブロック数だと超過分は未転記（ログに出る）。テンプレのブロックを増やす。
- **ファイルロック（重要）**：`02_output` 内の前回出力を **Word/Excel で開いたまま**だと、そのフォルダは上書きできず**スキップ**される（ログに「出力先のファイルが開かれています」）。実行前に出力ファイルを全て閉じる。`~$…` という一時ファイルが残っていたら開きっぱなしの証拠。

## 変更後にやること
- `99_data/パッチノート/パッチノート_vX.Y_内容_日付.md` を作成（何を・なぜ・検証結果・使用者が目視すべき点）。
- 使用者に「目視確認してほしい点」を明確に伝える（特に医療文言・様式の体裁）。

## 付記：この仕組みを新しいPCでも動かすには
- Python 3 と `openpyxl` `python-docx` が必要（`pip install openpyxl python-docx`）。
- フォルダ一式をコピーすれば動く（設定・テンプレ・スクリプトはフォルダ内に自己完結）。
- 実行は `転記実行.bat` をダブルクリック、または `py 99_data/src/transcribe.py`。
