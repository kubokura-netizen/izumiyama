# PRP申請書類 自動転記ツール（メンテナンス用）

クライアントは触りません。内部システム一式はこの `99_data/` に集約しています。

## フォルダ構成（UI向上版）
```
PRP申請書類 自動転記ツール/
├─ 転記実行.bat                         ← クライアントはこれをダブルクリックするだけ
├─ 00_仕様書・マニュアル/                ← 操作マニュアル等
├─ 01_input【ヒアリングシートをここへ】/  ← ヒアリングシート(.xlsx)を入れる
├─ 02_output【転記済みファイルがここに生成】/ ← 完成書類（全シート保持）
├─ 03_logs/                            ← 転記ログ
└─ 99_data/                            ← 内部システム（隠蔽）
   ├─ テンプレート/   … 2種関節系PRP.xlsx / 3種筋腱靭帯系PRP.xlsx / SOP.xlsx（17シート）
   ├─ マッピング/     … mapping.json（対応表の正本）/ format_changes.json（様式変更）
   ├─ src/           … transcribe.py / phase1_build_template.py / launcher.bas / 本書
   └─ 参考元【原本・設計書】/ … 原本ヒアリング・データマッピング・設計書
```
※ スクリプトは「自分の位置(99_data/src) → 親(99_data) → 親(ルート)」でパス解決。
   入出力フォルダは `01_input` / `02_output` の接頭辞一致で探すため、【説明】を付けても動く。

## 仕組み（5機能）
1. **複数シート完全出力**: `openpyxl.load_workbook → save` で全11シート・書式・画像を保持。
   特定シートだけ書き出さない（＝1シートしか出ない不具合を回避）。
2. **緑セル→マッピング**: `phase1` が両テンプレの緑フォントセルを走査し、`mapping.json` に
   各セルの転記元(source)を割り当て。未割当の緑セルは `source.t="unknown"` で残し、ログに
   「未割当(要マッピング)」として可視化（取りこぼし防止）。
3. **クロスシート検索（VLOOKUP的）**: `source.t` で対応。
   - `sheet`   : 医師略歴書 ← ヒアリングの「略歴書（PRP）」シートのセルを直接参照
   - `kit`     : 「PRPメーカー」シートをキット名で引く。`part` = 名称(A)/採血量(B)/採取(C)/加工(D)/**投与(E)**。
     - `idx:0` または `mode:"block"` → **選択された全キットを ①②③… で連番連結**（複数キット対応）。
       `with_name:true` で「①キット名＋本文」、`numbered:false` で連番なし、`joiner` で区切り変更。
       投与方法など全キット同一内容は自動で1つに畳む。
     - `idx:1,2,…` → そのキット単体（書類2の固定スロット用）。`with_mark:true` で先頭に①②…を付与。
   - `committee`: 認定番号 ← 「審査委員会」シートを委員会名で引く（委員会名↔認定番号の連動）。
   - `compose` : **複数項目を改行で連結**（「名称・所在地・連絡先が一行」問題の分割整形）。
     `lines:[{prefix,label,section}|{prefix,source}]` を上から `\n` で結合。
   - `doctor`  : **再生医療を行う医師（複数名 1〜15）**。`idx`=何人目, `field`=name/affiliation/role。
     ヒアリング「再生医療を行う医師…」で始まる人員欄のみ対象、雛形値(〇〇)は自動除外。
   - ※カテゴリ非依存の汎用ロジック。美容・育毛等へ流用する際もこの仕組みのまま使える
     （疾患名・治療名はヒアリング由来の変数のため差し替え不要）。
   - **未充足トークン掃除**: 出力直前にテンプレ内の残存 `{{…}}` を空欄化（`clear_tokens`）。
4. **フォルダ整理**: 実行ファイルと入出力以外を 99_data に隠蔽（上図）。
5. **オフライン様式変更対応**: `format_changes.json` を転記前に読み、`mapping.json` を動的に
   上書き（`apply_format_changes`）。label_renames / cell_moves / set_source / disable に対応。
6. **SOP（Word）差し込み**: `transcribe_sop.py` が `テンプレート/SOP/` フォルダを
   `02_output/SOP_(ヒアリング名)/` へ**フォルダごとコピー**し、各 .docx の定型値（クリニック名・
   氏名・制定日）をヒアリング値へ置換。図・表・書式はそのまま保持（python-docx使用・Word不要）。
   `転記実行.bat`（=transcribe.py）の実行で Excel書類と同時に出力される。

### SOP差し込みの設定（`マッピング/sop_replacements.json`）
- `template_folder`: テンプレフォルダ名（既定 `SOP`）, `output_prefix`: 出力名接頭辞（既定 `SOP_`）。
- `replacements[]`: `find`（テンプレ内の定型サンプル値・複数可）→ `source`（置換元）。
  - `source.t`: `hearing`（label/section/occ で索引）/ `today`（実行日。findに「年」が有れば年月日、無ければ月日）/ `fixed`。
  - 既定: クリニック名=「医療機関/名称」, 氏名=「特定細胞加工物製造事業者の名称」。**制定日は差し込まない**
    （テンプレの「制定：2026年〇月〇日」のまま手入力用に残す）。自動入力に戻すなら `today` ソースの設定を追記。
- 置換はラン単位→ラン跨ぎの2段で行い、書式を可能な限り保持。本文・表・ヘッダ・フッタを走査。
- ⚠ `.docx` のみ置換。`.dotx` 等はコピーのみ（中身そのまま）。python-docx 未導入時はSOPのみスキップ（Excelは出力）。
- ⚠ クリニックを変える際は、テンプレ側のサンプル値と `find` を合わせること（既定はサンプル=やまだ整形外科クリニック/山田 太郎）。

## よくある運用
- **項目名が変わった** → `format_changes.json` の `label_renames` に旧→新を追記（コード変更不要）。
- **様式でセルがずれた** → `cell_moves` に文書名・変数名・新セルを追記。
- **緑セルを足した/減らした** → `phase1_build_template.py` を実行して `mapping.json` を再生成。
  新しい緑セルは未割当として現れるので、`phase1` のキュレーション表に転記元を追記する。

## 既知の確認対象（要目視）
- 管理者氏名（医療法人=役職+氏名/個人=氏名のみ）、所属部署（欄なし→医療機関名代用）、
  事務FAX（欄なし）、医師役職（既定「院長」）、PRPキット採取/加工方法（PRPメーカー由来・体裁要整形）。
- 2種テンプレは様式1-2・略歴書・アセント以外の緑マークが未付与のため、それらシートは
  マーキング後に `phase1` 再生成で対象化できる（現状は3種が緑マーク充実）。
- 書類2の空の緑継続セルは「未割当」に出るが空セルのため転記不要。

## 実行
```
py 99_data/src/phase1_build_template.py   # mapping.json 再生成（任意・99_data/マッピング/へ）
py 99_data/src/transcribe.py              # 01_input の最新シートで転記 → 02_output
py 99_data/src/transcribe.py "パス\xxx.xlsx"   # ファイル指定
```
