# PRP 自動転記ツール ― ダッシュボード（ローカルWeb版）

`転記実行.bat`（コマンド操作）の代わりに、**ブラウザ画面から操作**できるようにしたものです。
既存の転記エンジン（`99_data/src/transcribe.py`）はそのまま呼び出すだけなので、
出力される書類の中身はこれまでと完全に同じです。

---

## 使い方（かんたん）

1. ツールのルートにある起動ファイルをダブルクリック
   - **Mac** → `ダッシュボード起動.command`
   - **Windows** → `ダッシュボード起動.bat`
2. 初回だけ準備が走ります（1〜2分）。自動でブラウザが開きます
   （開かない場合は `http://127.0.0.1:8765` を手入力）。
3. 画面で操作
   1. ヒアリングシート(.xlsx)を**ドラッグ&ドロップ**
   2. 一覧で対象ファイルを選び **「▶ 転記実行」**
   3. 進捗ログが流れ、完了すると **統計** と **要確認（黄色）項目** が表示
   4. 「📦 生成された書類フォルダ」から **ZIPでダウンロード**
4. 終了は、起動時に開いた黒い画面で **Ctrl + C**。

> Mac で「開けません（未確認の開発元）」と出たら、`ダッシュボード起動.command` を
> **右クリック → 開く** を一度だけ実行してください。

---

## 画面でできること

| 機能 | 説明 |
|---|---|
| アップロード | ヒアリングシートを `01_input` に保存（ドラッグ&ドロップ／クリック選択） |
| 転記実行 | `transcribe.py` を実行し、進捗をライブ表示 |
| 結果表示 | 転記済み／確認対象／未割当 の件数と、**要確認項目の一覧**（黄色セルに相当） |
| ダウンロード | `02_output` の各フォルダを ZIP でまとめて取得 |
| 実行履歴 | 過去の `03_logs` を一覧・再表示・Excelダウンロード |

---

## 仕組み（メンテ担当向け）

- `98_dashboard/app.py` … Flask の薄いラッパー。転記処理は **サブプロセスで transcribe.py を実行**するだけ。
- `98_dashboard/index_html.py` … 画面（1ページHTML、外部依存なし）。
- 実行後は `03_logs/転記ログ_*.xlsx` を解析して、`確認=="要確認"` の行を画面に出しています。
- **既存の src / マッピング / テンプレートには一切手を入れていません。** 転記ロジックの変更は
  従来どおり `99_data/マッピング/format_changes.json` 等で行えます。

### ポート番号を変えたいとき

環境変数 `PRP_DASH_PORT` で変更できます（既定 8765）。

### 手動で起動する場合

```
python 98_dashboard/app.py
```

（`flask` `openpyxl` `python-docx` が必要。`98_dashboard/requirements.txt` 参照）

---

## 注意

- このダッシュボードは **自分のPC内だけ**で動きます（`127.0.0.1`）。
  社外・ネット越しにはアクセスできません＝医療情報は外に出ません。
- 同時に複数人が別々のPCで各自起動して使う想定です（1台を共有サーバにはしていません）。
