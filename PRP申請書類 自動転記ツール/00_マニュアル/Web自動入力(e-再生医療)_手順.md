# 🌐 Web自動入力（e-再生医療）手順 ＝ 方式B（半自動）

e-再生医療（`https://saiseiiryo.mhlw.go.jp/application/plan01`）のWebフォームへ、**アウトプット（02_output の転記済み資料）の値を自動で入力**する補助ツールです。

> **重要**
> - このツールは**入力（下書き）まで**を自動化します。**送信は必ず人が最終確認して行ってください**。
> - **参照元はアウトプット優先**：出力様式xlsxのセル（`cell`）→ 出力Wordの見出し（`docx`）→ 無ければヒアリング（`source`）の順。手直しした最終値がそのまま入ります。
> - **このサイトはログイン不要**です（ログイン待ちの手順はありません）。
> - 政府ポータルの自動操作は**利用規約の確認が前提**です。

---

## ★ かんたん起動：`web転記実行.bat`（ダブルクリック）

ツールのフォルダにある **`web転記実行.bat`** をダブルクリックすると、メニューが出ます：
```
  1 : Auto-fill   … 自動入力する（本番）
  2 : Dump fields … フォーム項目を抽出する（対応表づくり）
  3 : Setup       … 初回準備（Playwright 導入）
  Q : Quit        … 終了
```
- **初回**：まず `3`（Setup）→ 導入完了後 `Q`。
- **対応表づくり**：`2`（Dump）。
- **本番の自動入力**：`1`（Auto-fill）。
- ※ メニューは英語表記ですが、操作は数字を打って Enter するだけです（日本語をbatに入れると文字化けするため英語にしています）。

以下は中身の詳しい説明です。コマンドで直接動かす場合の参考にもなります。

## 0. 準備（初回のみ）＝ メニューの `3`

コマンドで行う場合（Playwrightという自動化ライブラリと、専用ブラウザを入れます）：
```
pip install playwright
playwright install chromium
```

---

## 1. フォームの入力欄を抽出する（対応表づくりの下調べ）

```
py 99_data\src\web_fill.py --dump
```
1. 専用ブラウザが開くので、**ログインして対象フォームを表示**。
2. コマンドプロンプトに戻って **Enter**。
3. → `03_logs\web_fields_dump.txt` に、フォームの入力欄一覧（**selector候補**とラベル）が出力されます。

---

## 2. 対応表を作る（`99_data\マッピング\web_mapping.json`）

`web_fields_dump.txt` を見ながら、各項目の **`selector`（またはラベル）** と入力値を埋めます。
`source`（どのヒアリング項目を入れるか）は既に例が入っています（必要なら調整）。

### 欄の種類ごとの書き方
```jsonc
// ① 通常のテキスト / 長文テキストエリア
{ "desc":"医療機関名", "selector":"#hospitalName", "type":"text",
  "source":{ "t":"hearing", "label":"医療機関/名称（診療所開設届上）", "section":"法人/医療機関" } }

// ② ラジオ（有/無・該当/非該当）… choice に「選ぶ選択肢の文言」を書く
{ "desc":"役務の提供の有無", "selector":"input[name='xxx']", "type":"radio", "choice":"無" }

// ③ プルダウン
{ "desc":"○○区分", "selector":"#kbn", "type":"select",
  "source":{ "t":"hearing", "label":"…", "section":"…" } }

// ④ 採取/加工/投与方法・治療価格・採血量 など（トークンで解決）
{ "desc":"採取方法", "selector":"#saishu", "type":"text",
  "source":{ "t":"token", "name":"採取方法" }, "saisei":1 }

// ⑤ selectorが不安定なとき → label（設問文言）で特定
{ "desc":"役務提供者の名称", "label":"役務提供者の名称", "type":"text",
  "source":{ "t":"hearing", "label":"…", "section":"…" } }
```
- `type`：`text`（テキスト/テキストエリア共通）/ `select` / `radio` / `checkbox`。
- **radio** は `choice` に選ぶ選択肢の文言（有/無/該当/非該当）を書く。`source` で値から決めることも可。
- `source.t`：`hearing`/`committee`/`doctor`/`fixed` のほか、`token`（採取方法・加工方法・投与方法・治療価格・必要な採血量 等を汎用解決）。`saisei` は 2種=1／3種=2。
- `selector` も `label` も空の欄はスキップされます。

---

## 3. 自動入力する（複数セクション対応）

```
py 99_data\src\web_fill.py
```
1. ブラウザが開く（**ログイン不要**）→ plan01フォームを表示。
2. コマンドプロンプトで **Enter** → 参照アウトプット名が表示され、該当欄が自動入力＆**緑枠でハイライト**。
3. 複数ページなら画面の「**次の項目へ**」で次を表示 → **Enter** で続けて入力。
4. 全ページ終わったら **q + Enter** で終了。
5. **内容を確認して、最後にあなたが送信（入力内容確認→送信）してください**（ツールは送信しません）。

> どのアウトプットを参照するかは `web_mapping.json` の `output_folder_contains`（既定=2種関節系PRP）で決まります。02_output内の該当フォルダの最新が使われます。3種は `3種筋腱靭帯系PRP` に変更。

> ヒント：各セクションで「入力○件」のログが出ます。「見つからない」欄は別ページにあるか、selector/labelの見直しが必要です。

---

## 補足・注意
- **ログイン状態は維持**されます（`99_data\_web_profile` にプロファイル保存）。次回はログイン省略できる場合があります。共有PCでは注意。
- **見つからない欄**は「見つからない: …」とログに出ます → selector を見直す（画面遷移で動的に出る欄は、表示してからEnter）。
- **複数ページ**のフォームは、ページごとに「表示→Enter」を繰り返す運用にできます（必要なら拡張します）。
- サイト改修で欄の構造が変わると selector の更新が必要です。
- 入る値は今のヒアリングシート（`01_input`）の内容です。事前に `転記実行.bat` 等で値が正しいか確認しておくと安心です。
