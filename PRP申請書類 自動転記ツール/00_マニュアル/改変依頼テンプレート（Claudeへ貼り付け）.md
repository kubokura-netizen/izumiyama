# 🛠 ツール改変 依頼テンプレート（Claude Code へ貼り付けて使う）

このツール（PRP申請書類 自動転記ツール）を**壊さずに安全に改変**するための指示書です。
プログラミングの知識がなくても、下の「**■ Claudeへの指示**」を丸ごとコピーして、
**【変更内容】** に自分のやりたいことを書き、**Claude Code に貼り付ける**だけで改変できます。

このツールには **2つの機能** があります。どちらの話かを Claude が判断できるよう、
【変更内容】には「**書類転記のこと**」か「**Web転記のこと**」かを一言添えると確実です。

| 機能 | 何をする | 実行入口 | 設定ファイル |
|---|---|---|---|
| **① 書類転記ツール** | ヒアリングシート → 申請書類一式(Excel/Word)を生成 | `転記実行.bat` / ダッシュボード | `99_data/マッピング/docs_config.json` |
| **② Web転記ツール** | 生成した書類の内容 → e-再生医療フォームへ自動入力＋一時保存＋添付 | `web転記実行.bat` / ダッシュボード | `99_data/マッピング/web_mapping.json` |

> ②は①の出力（`02_output`）を読んでWebに入れます。つまり **書類転記 → Web転記** の順に使います。

---

## 📌 使い方（3ステップ）

1. 下の「**■ Claudeへの指示（ここからコピー）**」〜「**（ここまでコピー）**」を**丸ごとコピー**。
2. `【変更内容】` の下に、**やりたい変更を日本語で具体的に**書く（例文は付録B・付録E）。
3. このツールのフォルダを開いた **Claude Code** に貼り付けて送信。

> 💡 Claudeが自動で「今の作りを調べる → バックアップ/コミット → 変更 → テストして検証」まで行います。
> 変更後は必ず、①なら `転記実行.bat`、②なら Web転記（一括）を一度実行し、結果を目視で確認してください。

# 付録A：書類転記ツール（①）の全体像

## フォルダ構成
```
PRP申請書類 自動転記ツール/
├─ 転記実行.bat                         … ①実行入口（py transcribe.py を呼ぶ）
├─ web転記実行.bat                       … ②実行入口（py web_fill.py を呼ぶ）
├─ ダッシュボード起動.bat                … ①②をブラウザUIから操作（Flask）
├─ 00_マニュアル/                        … 本書・操作マニュアル・Web手順
├─ 01_input【…】/                        … ヒアリングシート(.xlsx)を置く（①の入力）
├─ 02_output【…】/                       … ①の出力（フォルダ単位で生成）＝②の入力元
├─ 03_logs/                             … 転記ログ・Web転記のレポート/スクショ
├─ 98_dashboard/                        … app.py（Flask）＋ index_html.py（画面）
└─ 99_data/
   ├─ テンプレート/                       … 2種/3種/SOP の様式xlsx＋docx＋PDF一式
   ├─ マッピング/
   │   ├─ docs_config.json              … ★①転記設定の本体（最重要）
   │   ├─ web_mapping.json              … ★②Web入力の対応表（最重要）
   │   ├─ mapping.json                  … 様式セル転記の元定義
   │   └─ …（format_changes.json 等）
   ├─ src/
   │   ├─ transcribe.py                 … ①main。ヒアリング読込→run_docs呼出→ログ
   │   ├─ transcribe_docs.py            … ①フォルダ型エンジン（様式Excel＋docx置換＋医師/略歴書/キット）
   │   ├─ transcribe_sop.py             … ①Word置換の部品
   │   ├─ web_fill.py                   … ★②Web入力エンジン（Playwright）
   │   └─ README_メンテナンス.md
   └─ パッチノート/                       … 変更履歴（改変したら追記）
```

## ①処理の流れ
`転記実行.bat` → `transcribe.py main()` → ヒアリング読込 → `transcribe_docs.run_docs()`：
各テンプレフォルダ（2種/3種/SOP）を `02_output/(フォルダ名)_(ヒアリング名)/` へコピーし、
1. **様式第一の二(.xlsx)** … `docs_config` の `excel.entries`（セル→ヒアリング項目）でセル転記＋
   `doctor_fill`（医師を最大15名、ブロックを動的検出して転記）。
2. **各docx** … `docs_config` の `docx`（サンプル値→ヒアリング値の一括置換）＋
   `docx_kit`（`{{採取方法}}`/`{{加工方法}}` を①②③連番展開）＋
   `docx_table`（略歴書を表の行ごとに転記）。
3. **PDF・論文・追加xlsx** … 無変更で同梱。

## docs_config.json の source（転記元）
| t | 意味 | 例 |
|---|---|---|
| `hearing` | ヒアリングのラベル＋セクションで検索 | `{"t":"hearing","label":"…","section":"…"}` |
| `sheet` | ヒアリング内の別シートのセルを直接参照 | `{"t":"sheet","sheet":"略歴書（PRP）","cell":"B2"}` |
| `committee` | 委員会名→審査委員会シートで認定番号 | `{"t":"committee"}` |
| `kit` | PRPメーカーシートから採取/加工/投与。`mode:"block"`で①②③連番 | `{"t":"kit","part":"加工","mode":"block","numbered":true}` |
| `doctor` | 医師（複数）※通常は doctor_fill を使う | `{"t":"doctor","idx":2,"field":"name"}` |
| `date` / `fixed` / `unknown` | 実行日 / 固定値 / 未定義 | |

# 付録B：書類転記（①）変更タイプ別 早見表

> 補足：規制上の手続き（重要変更＝様式第2で委員会審査 等）は**使用者の申請業務**です。
> ここでは「**このツールの出力を変える**」ための改変方法を示します。

| 変更したいこと | 主な対応（設定優先） |
|---|---|
| **様式改訂で項目が追加** | テンプレ(.xlsx/.docx)に欄を用意 → `docs_config` に転記設定を追加 |
| **様式改訂で項目が削除** | 該当の転記設定を削除/無効化＋テンプレから欄を削除 |
| **医師の追加・変更** | ヒアリングの医師欄に追加すれば `doctor_fill` が最大15名自動転記。テンプレのブロックが足りなければ様式Excelで医師ブロックを増やす |
| **PRPキットの変更・追加** | PRPメーカーシートに行を追加（A:名称/B:採血量/C:採取/D:加工/E:投与）。書類2は `{{採取方法}}{{加工方法}}` で自動連番展開 |
| **提供価格の変更** | ヒアリング項目化して転記 or `docx` の find-replace or テンプレ直接編集 |
| **軽微な文言変更** | `docs_config` の `docx`（find→source）に置換を追加、またはテンプレを直接編集 |

# 付録C：書類転記（①）の検証・落とし穴

## 検証（変更後は必ず）
```
py 99_data/src/transcribe.py            # 01_input の最新ヒアリングで実行
py 99_data/src/transcribe.py "パス.xlsx"  # ファイル指定で実行
```
- **出力フォルダ名**が `2種関節系PRP_(ヒアリング名)` 等になっているか（壊れた長い名前＝変数衝突を疑う）。
- 変更項目が反映され、旧サンプル値（やまだ整形外科クリニック／山田太郎 等）が残っていないか。
- **文字化け対策**：Windowsコンソールは cp932 で日本語が化ける。検証は UTF-8 テキスト出力や、
  docx は `<w:t>…</w:t>` の可視テキスト抽出で行う（属性のparaId等を数えないこと）。

## よくある落とし穴（①）
- **セル座標のズレ**：2種と3種は行構成がずれる箇所がある。種ごとに確認。
- **置換対象を割らない**：テンプレの「置換される値」「{{トークン}}」の途中に改行・スペースを入れない。
- **テキストボックス内の文字**：Wordの図形内は低レベル `<w:t>` 走査で対応済み。
- **変数名の衝突**：`transcribe_docs.py` でファイル名=`fbase`、ヒアリング名=`base`。混同すると出力名が壊れる。
- **ファイルロック**：`02_output` の前回出力を Word/Excel で開いたままだとスキップされる。実行前に閉じる。

# 付録D：Web転記ツール（②）の全体像と改変ガイド

## これは何か
生成した書類（`02_output`）の内容を、**e-再生医療の申請フォーム(plan01)へ自動入力**し、
**一時保存（下書き保存）→受付番号取得→添付書類のアップロード**まで行う補助ツール。
**送信は絶対にしない**（最終確認と送信は必ず人）。Playwright でブラウザを自動操作します。

## 実行方法
- **ダッシュボード**：`ダッシュボード起動.bat` → Web転記カード →「🚀 一括入力を実行」（推奨）
- **bat**：`web転記実行.bat`
  - `A` … 一括（入力→一時保存→添付→再保存）＝本番
  - `2` … Dump（フォームの項目/selectorを抽出。web_mapping メンテ用）
  - `T` … 添付だけ試す（--attach。保存済み下書きを開いた状態で実行）
  - `3` … 初回準備（Playwright導入）
- **コマンド**：`py 99_data/src/web_fill.py --auto` / `--dump` / `--attach`（`--only=2` で1件、`--no=受付番号` で下書き指定）

## 参照元の優先順位（②の入力データ）
`02_output` の最新の該当フォルダ（`output_folder_contains` で絞る）を読み、各欄の値を決める：
1. **`row`（見出し参照）** … 様式Excelを「見出し文字列」で探して値を読む【推奨・最優先】
2. `cell`（行番号）… 非推奨（行がズレると壊れる。現在は使っていない）
3. `docx` … 出力Wordの見出しセクション
4. `source`（ヒアリング）… フォールバック
5. `choice`/`fixed` … 固定の選択・固定値（参照元不要）

## web_mapping.json の構造
```jsonc
{
  "url": "https://saiseiiryo.mhlw.go.jp/application/plan01",
  "output_folder_contains": "2種関節系PRP",   // どの案件フォルダを参照するか（3種は変更）
  "output_sheet": "1 の２提供計画（治療）",

  // 一括モードの制御
  "tabs": ["申請者情報","項目1", … ,"項目7"], // タブ式フォーム。順に開いて入力する
  "save_selector": "", "next_selector": "",    // 空なら文言で自動探索

  // 文字数オーバー対策（Webは改行を\r\n=2文字で数える）
  "char_limit_default": 4000, "char_warn_default": 3200,
  "overflow_mode": "mark",       // mark=本文を入れず「文字数超過」と記入して保存を通す
  "overflow_mark": "文字数超過",  //   （truncate=末尾を…で切る / report=そのまま入れる）

  // 添付（一時保存後にのみ可能。#btn_upload_fileupload<N> の N=書類番号 に直接セット）
  "attach_enabled": true,
  "attach_slot_map": { "01": "1", "4.5": ["4","5"] }, // 番号→スロット。リストで同一ファイルを複数欄へ
  "attach_login_url": "https://saiseiiryo.mhlw.go.jp/application/login/plan",

  "fields": [
    // 例1: 見出しで本文を読む（行がズレても追従）
    { "desc":"安全性についての検討内容", "selector":"#safe_safety", "type":"text",
      "row": { "heading":"提供する再生医療等の安全性についての検討内容", "col":"L" } },

    // 例2: チェックボックス（固定でON）。type は必ず書く（後述の hidden 対策）
    { "desc":"特定細胞加工物(チェック)", "selector":"input[name='data[Plan][cells_type1]']",
      "type":"checkbox", "source":{ "t":"fixed", "value":"1" } },

    // 例3: ラジオ（固定で選択）
    { "desc":"再生医療等の分類", "selector":"input[name='data[Plan][typing]']",
      "type":"radio", "choice":"第二種" },

    // 例4: ■□マーカーで有無を判定（見出し行の L列=有側 / R列=無側 を見る）
    { "desc":"補償の有無(細胞提供者)", "selector":"input[name='data[Plan][cell_donor_reparation]']",
      "type":"radio",
      "choice_from":{ "heading":"補償の有無", "occ":1, "options":[["有","L"],["無","R"]], "marked":"■" } },

    // 例5: 日付を年/月/日の3プルダウンに分解（日付型/文字列/シリアル値どれでもOK）
    { "desc":"意見書の発行日 年", "selector":"#PlanOpinionIssueDateYear", "type":"select",
      "row": { "heading":"認定再生医療等委員会による意見書の発行日", "col":"L", "tf":"date_year" } }
  ]
}
```

## row（見出し参照）の指定オプション
| キー | 意味 |
|---|---|
| `heading` | 探す見出し文字列（部分一致） |
| `col` | 値がある列（通常 "L"、様式ヘッダは "N"） |
| `exact` | true で見出し完全一致（「所属機関」が「所属機関の郵便番号」に誤ヒットするのを防ぐ） |
| `occ` | 同じ見出しが複数ある時、何個目か（実施責任者/事務担当者の「氏名」等） |
| `offset` | 見出し行から何行ずらすか（負も可。値が見出しの上にあるヘッダ部用） |
| `span` | 読む行数を固定（1=その1行だけ。医師の氏名/所属など） |
| `head_cols` | 見出しを探す列（既定 B/C/D）。ヘッダ部は ["L"] を指定 |
| `tf` | 変換：`date_year/date_month/date_day`（日付分解）/ `pref`(都道府県) / `addr_body`(住所) / `zip` |

## ②変更タイプ別 早見表
| やりたいこと | 対応 |
|---|---|
| **Web欄に別のExcel見出しの値を入れたい** | その欄の `row.heading` を変える（`--dump` で selector を確認） |
| **固定でチェック/選択したい** | checkbox は `source:{t:fixed,value:"1"}`、radio は `choice:"…"` |
| **有無を Excel の■/□で自動判定** | `choice_from:{heading,options:[["有","L"],["無","R"]],marked:"■"}` |
| **文字数オーバーの扱いを変える** | `overflow_mode`（mark/truncate/report）。欄ごとに `char_limit` 上書き可 |
| **添付の貼り先を変える／複数欄へ** | `attach_slot_map`（例 `{"4.5":["4","5"]}`＝同じファイルを書類4と5へ） |
| **添付する拡張子を増やす** | `attach_exts`（既定 docx/xlsx/pdf）。除外は `attach_skip_prefix` |
| **3種に切り替え** | `output_folder_contains` を "3種筋腱靭帯系PRP" に。委員会構成は type1→type2、疾患名等も要調整 |
| **フォームの構造が変わった** | まず `--dump` で新しい selector を確認 → `fields` の selector/heading を更新 |

- 「**Web転記**：4.5の説明書を書類4と書類5の両方に添付して。」
- 「**Web転記**：この欄はオーバーしても本文を切って入れたい（truncate）。」

# 付録E：Web転記（②）の検証手順・落とし穴・教訓

## よくある落とし穴（②）と、これまでの教訓
- **同名の hidden input（最重要）**：CakePHP系フォームは checkbox/radio の直前に同じ name の
  `<input type="hidden">` を出力する。selector に type を書かないと hidden を掴んでスキップされる
  （＝チェックが入らない）。→ web_fill 側で type を自動付与するが、**マッピングでも type を必ず書く**。
- **カスタム装飾のチェック**：値を直接代入してもサイトが認識しない。**可視ラベルを実クリック**する
  （web_fill 実装済み）。「入って消える」時はクリックの二重発火を疑う。
- **添付は一時保存後のみ**：添付欄は「一時保存した下書き」でしか描画されない。順序は
  入力→一時保存→保存データ編集に戻る→添付→再度一時保存（--auto はこの順で自動）。
- **添付は input[type=file] に直接セット**：`#btn_upload_fileupload<N>`（N=書類番号）へ set_input_files。
  「ファイル選択」は装飾で、file_chooser も不要。`.first` だと書類1に貼られるので必ず番号で掴む。
- **行番号(cell)は使わない**：転記ツール改修で出力Excelの行が増減すると壊れ、しかも黙って別の値が入る。
  必ず `row`（見出し参照）を使う。既存の cell が残っていたら row に移すのが安全。
- **文字数はWeb基準で**：Webは改行を `\r\n`=2文字で数える。ツールも改行を2文字として計上する。
- **別タブの要素に触ると固まる**：非表示の select/text に操作すると待ち時間で数十秒フリーズする。
  可視でなければ「後回し」にする（実装済み。新規で触るときは同様に）。

## 付記：新しいPCでも動かすには
- 共通：Python 3 と `openpyxl` `python-docx`（`pip install openpyxl python-docx`）。
- ②Web転記：さらに `playwright`＋Chromium（`pip install playwright` → `playwright install chromium`。
  bat の `3 : Setup` でも導入可）。
- パスは全て相対解決（`02_output` は前方一致）なので、フォルダ一式をコピー／クローンすれば誰のPCでも動く。
  絶対パスやユーザー名はコードに書かない（環境依存を作らない）。
