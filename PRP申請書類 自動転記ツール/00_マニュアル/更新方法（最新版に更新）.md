# 🔄 ツールの更新方法（最新版に更新）

開発（大野／窪倉）が GitHub に push した内容を、クライアントのPCに**ダブルクリック1回**で反映するための仕組みです。

---

## 👤 クライアントの操作（かんたん）

ツールのフォルダにある **「最新版に更新.bat」** をダブルクリックするだけです。

1. 黒い画面が開き、「最新ファイルをダウンロードしています…」と進みます（数分・ネット接続が必要）。
2. 「✔ 最新版に更新しました。」と出たら完了。Enter キーで閉じます。
3. あとはいつも通り **「ダッシュボード起動.bat」** をダブルクリックして使えます。

> - **患者情報（01_input / 02_output / 03_logs）や、ログイン情報、同梱のPython(_runtime)はそのまま残ります。**上書きされるのはツールのプログラム部分だけです。
> - 「WindowsによってPCが保護されました」と出たら「詳細情報」→「実行」。
> - 失敗したときは、ネット接続を確認してもう一度。直らなければ画面のまま開発担当へ連絡してください。

---

## 🛠 開発側（大野・窪倉）の運用

**特別な作業は不要です。いつも通り push するだけ**で、クライアントが次に「最新版に更新.bat」を押した時点で最新になります。

- リポジトリ: `kubokura-netizen/izumiyama`（**public**）の `main` ブランチ。
- パッチャーは `codeload` から **常に main の最新** を取得するため、
  **どちらが push しても**、ビルドやリリース作業なしにそのまま反映されます。
- 大きな変更を入れたら、念のため自分のPCで「最新版に更新.bat」を1回試すと安心です。

### 仕組み（ファイル）
| ファイル | 役割 |
|---|---|
| `最新版に更新.bat`（ツール直下） | クライアントがダブルクリックする入口（中身はPowerShellを呼ぶだけ） |
| `98_dashboard/_update.ps1` | 本体。GitHubのmain.zipを取得→展開→コードを上書き→部品を確認 |

- **保持（上書きしない）**: `01_input` / `02_output` / `03_logs` / `_runtime` / `.venv_dashboard` / `99_data/_web_profile`
- **更新**: `98_dashboard` / `99_data/src` / `99_data/マッピング` / テンプレート / 各種`.bat` / マニュアル
- 更新後、`_runtime`（または`.venv_dashboard`）に不足している部品（playwright/pywin32/PyMuPDF等）があれば**自動で導入**します。

### 初回のクライアント配布
初回だけは Python同梱の配布物が要ります（更新では `_runtime` を配れないため）。
1. `98_dashboard/配布ランタイム作成.ps1` を実行（`_runtime` を作成）
2. `98_dashboard/配布ZIP作成.ps1` を実行（`最新版に更新.bat` も自動で同梱されます）
3. できたZIPをクライアントへ。以降の更新は「最新版に更新.bat」だけでOK。

### 注意
- 実行中の `最新版に更新.bat` 自身は更新対象から除外しています（cmd破損防止）。
  この入口ファイルの中身を変えた場合のみ、次回配布ZIPで配り直してください（通常は変更不要）。
- `_update.ps1` は日本語を含むため **UTF-8(BOM付き)** で保存すること（BOM無しだと文字化けで動きません）。
